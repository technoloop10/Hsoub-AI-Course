

## 1. البرمجة كائنية التوجه (OOP)
نمط برمجى يعتمد على تنظيم الكود في وحدات تسمى **الأصناف (Classes)** و**الكائنات (Objects)**.

### 2. معايير تسمية المعرفات (Naming Conventions)

يتبع المبرمجون في بايثون دليل التنسيق (PEP 8) لتوحيد تسمية العناصر:

#### **الأنماط المستخدمة:**
* **PascalCase:** يبدأ كل جزء من الاسم بحرف كبير بدون فواصل.
    * **الاستخدام:** مخصص لتسمية **الأصناف (Classes)** فقط.
    * *مثال:* `Product`, `ElectronicItems`.
* **snake_case:** جميع الحروف صغيرة مع استخدام الفاصلة السفلية `_` بين الكلمات.
    * **الاستخدام:** مخصص لتسمية **الكائنات (Objects)**، **التوابع (Methods)**، و**المتغيرات**.
    * *مثال:* `iphone_14`, `apply_discount`.

#### **القيود البرمجية في التسمية:**
* **المحاذير التقنية (تسبب خطأ في التنفيذ):**
    * بدء الاسم برقم.
    * استخدام المسافات أو الرموز الخاصة (`!`, `@`, `#`, `-`).
    * استخدام الكلمات المحجوزة في لغة بايثون (مثل `class`, `def`, `while`).
* **الأنماط غير القياسية:**
    * **camelCase:** يبدأ بحرف صغير ثم حرف كبير (مثل `myProduct`). برمجياً هو جائز، لكنه لا يتوافق مع معايير بايثون القياسية (Standard).

### 3. تطبيق عملي: صنف المنتج (Product Class)

```python
class Product:
    # دالة البناء (Constructor): لتعريف الخصائص الأولية
    def __init__(self, id, name, price, count):
        self.id = id          # معرف المنتج
        self.name = name      # اسم المنتج
        self.price = price    # السعر
        self.count = count    # الكمية

    # تابع (Method): دالة معرفة داخل الصنف لتنفيذ عملية محددة
    def discount(self, ratio):
        self.price = self.price - (self.price * ratio)

# إنشاء كائنات (Objects)
iphone_14 = Product(1, 'iPhone 14', 999, 10)
samsung_s23 = Product(2, 'Samsung S23', 850, 5)

# استدعاء التابع (Method Call)
iphone_14.discount(0.1)


. المفاهيم التقنية للمثال:
الخصائص (Attributes): هي المتغيرات (id, name, price, count) التي تخزن حالة الكائن.
التوابع (Methods): هي الدوال المعرفة داخل الصنف (مثل discount). تختلف عن الدوال المستقلة (Functions) في أنها تتطلب وسيط self للوصول إلى بيانات الكائن الذي قام بالاستدعاء.
المعامل self: مرجع يشير إلى الكائن الحالي، يُستخدم لربط البيانات والعمليات داخل الصنف.
